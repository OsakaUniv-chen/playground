# 博论主线速录 — Proposal 2（"能動性(proactivity)"为主题）

> proposal1 的替代叙事。**主题＝proactive robot（能動的ロボット）**，这是目的。
> proactivity 有两个侧面：**能動的行動**（P1-2，核心）与**能動的注意/知覚**（P3-6，扩展）。
> 音图＋预测（PSSP）是让注意变 proactive 的**手段**，不是目的；"知道相手是谁"是手段的即时功能。
> 对照版：[../proposal1/mainline-notes.md](../proposal1/mainline-notes.md)。最后更新：2026-07-16

---

## 1. 主题（thesis theme）

> **実世界の多人数 HRI で、反応的ではなく"能動的（proactive）"に振る舞う社会ロボットを
> どう実現するか。**

能動的＝**明示的トリガーを待たずに、先んじて**行動・注意を起こす。这个主题下有两个侧面：

- **能動的行動（proactive action）**：适时自发起动社会行动（P1-2，**已达成、超越人类**）。
- **能動的注意/知覚（proactive attention）**：要让行动先于事件，**知覚も先んじる**必要——
  预测"现在/接下来相手是谁"。视觉不够 → 音图＋预测（PSSP）是手段（P3-6）。

**anticipation（予測）是 proactivity 的下位机制**：能動が目的、予測が手段。
这也厘清了 proposal1 里 proactive/anticipatory 概念重叠的问题。

## 2. 叙事骨架（proactivity 如何贯穿五篇）

| 研究 | 在"能動性"主题下的角色 | 结果与局限 |
|---|---|---|
| **P1** | **能動的行動の実現**：商场 android，7 预设动作，遥操作者/IRL 从 7 动作选一（2Hz），机器人**自发**在对时机起动作 | 接近人操作。能動的行動可行 |
| **P2** | **能動的行動が人を超える**：同实验改 RL | **超越人类**。论文旗舰成果 |
| **——扩展的动机——** | 要把能動性扩展到**注意**：多人场景中机器人得先知道"相手是谁"才能先行动。但 P1-2 的 TRACK 只看**最近的脸**≠说话者，且**口罩**令视觉失效 | → 需要**非视觉、且能先読み**的知覚手段 |
| **P3** | **能動的注意の手段を作る**：sound map generation ＋ 预测模型 **PSSP**。用声音定位相手（不惧口罩）＋**预测**未来音图以**先んじて**转向（减少视线延迟＝注意的能動化） | 只评估**预测性能**，未评估其对人**知觉**的影响 |
| **P4** | **能動的注意は効くか（初探）**：word wolf，4 注视目标选择（Tele/PSSP/DoA/Random），※目标与运动**捆绑** | n=2 小样本，GQS/GEQ 测出差异 PSSP>Tele>DoA>Random |
| **P5** | **能動的注意の厳密検証**：word wolf 13 组，**解耦目标与运动**（只输出左/右、运动共通） | 运动学有差异，但知覚（GQS/GEQ）**几乎无差** → 见 §3 |
| **P6** | **能動的行動＋能動的注意の統合を、有後果の现场で**：くじ引き（见设计稿） | 待做 |

**一句话**：P1-2 让机器人**行动**上能動（且超越人类）；P3-6 试图把能動性扩展到**注意**
（预测为手段），并厘清它何时真有用。

## 3. P4 vs P5 的分辨（必须当场讲清）

- P4 的 PSSP 优势来自**捆绑体（目标＋运动）**；P5 运动共通、只变目标后优势消失。
- → **P4 测出的知觉差异来自运动分量，而非注视目标的选择**。P5（powered＋解耦）是可信版本。
- P4 n=2，"PSSP>Tele"等排序不可过度当真。

## 4. 主张层（贡献命题——骨架之上必须有）

主题是"如何实现能動的ロボット"，本论文的**贡献命题**（proactivity 主题下、对 P6 正负都成立）：

> 本论文**实现**了能動的な社会ロボットの两个侧面——**行動**（P1-2，超越人类）与
> **注意/知覚**（P3，预测的音源注意）——并**厘清了能動的注意が実際に効く境界条件**
> （P4-6：孤立的先読み注视不改善知覚，改善的是运动/需以任务后果衡量）。

- 这个命题让 **P1-2 稳居核心**（能動的行動的旗舰实现），P3-6 是能動性向注意的扩展与其边界刻画。
- P5 的 null 在此框架下是"**能動的注意の境界の一点**"，非跑题的失败。

## 5. 诚实的软肋（绝对客观）

1. **隐藏假设（上一轮暴露，最优先）**：proposal2 的因果链暗含"给已超越人类的 P2 换上 PSSP 注视
   → 整体更好"。但 (a) 这与 **P5 直接冲突**（运动共通时换更准的注视目标，知觉不动）；
   (b) **P2 的超越根本不靠注视**（注视只是七动作里粗糙的 TRACK，超越来自动作时机策略）——
   所以"注视是 P2 的瓶颈"这个先验**没有证据、甚至反证**。
   → 对策：**把这个假设摆上台面当被检验命题**，且把 P6 的问题从"能否让 P2 更好"（赌赢面小、
   null 难看）**降级为边界刻画**："选对相手的能動的注意，**在什么场景才成为任务瓶颈**？"
   （P1-2 的单人为主商场里不是瓶颈；多人竞争性场景才可能是——P6 去找那条边界）。
   这实质是把 proposal1 的"价值条件"思想塞回来当安全网。
2. **日记风险**：骨架是时间线，靠 §4 主张层兜住"贡献命题是什么"。
3. **P5 null 仍是中心，仍需 P6 landing**：主张不能是"PSSP 解决了问题"，只能是"厘清了边界"。
4. **Lag 框架的张力**：PSSP 若本质是"减少滞后/更早转向"，P5 恰好直接检验并给出负结果
   （更早的注视不改善知觉）。残留问题：预测是否另有"运动更平滑"的收益被 P5 共通控制器抹平
   （P4 的暗示）——可留给 P6/future，但**别把 P5 说成"没测对东西"**。
5. **架构断层**：注意从 P1-2 离散动作库里的一个选项（TRACK）变成 P3-5 的独立连续通道——
   proposal1 §9 已记录，proposal2 里更需明说（也关系到 P6「統合」到底是深融合还是接口）。

## 6. 相对 proposal1 的定位（客观）

- **proposal1**：两个 Field（行動/知覚）并排、P6 缝合；抽象价值矩阵；banner 贡献式。
- **proposal2（本版）**：**单一主题 proactivity**，行動（核心）与注意（扩展）是它的两个侧面，
  预测为手段；P1-2 居中心，连接靠"多人注意需要先読み知覚"的具体动机。
- 判断：本版比 proposal1 **更统一、P1-2 更强、动机更具体**；代价是 P5 的 null 与 §5-1 的隐藏
  假设更需要用"边界刻画"（proposal1 的思想）来兜。→ **终局仍可能是混合**：proposal2 的
  proactivity 主题＋具体骨架，吸收 proposal1 的"境界条件"作为 §4 主张与 P6 的安全网。
