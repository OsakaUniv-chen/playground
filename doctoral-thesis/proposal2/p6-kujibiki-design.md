# P6 くじ引きロボット実験 設計草案 — Proposal 2（"交互对象"框架）

> 差别在**framing**：P6 是"把**能動的行動＋能動的注意**统合进一台机器人，在有后果的现场
> **刻画能動的注意的价值边界**"。

## 1. P6 在 proposal2 叙事里的位置：能動性の統合と境界検証

- **P1-2 = 能動的行動**（超越人类）；**P3-5 = 能動的注意の手段（PSSP）と隔离环境での検証**
  （word wolf；P5 发现孤立的先読み注视不动知覚）。
- **P6 = 把两个侧面统合进一台机器人**，放到 P1-2 那类**现实・有后果**的场景，检验
  **能動的注意在什么场景才成为任务瓶颈**（**境界刻画**，不是"能否让系统更好"——见
  mainline §5-1 隐藏假设）。
- **くじ引き**：机器人在人来人往中**先んじて**招呼该招呼的人、递くじ。
  谁被（能動的に）注意到 → 谁被招呼 → 谁拿到くじ。注意的能動性由此获得任务后果。

**为什么是边界刻画而非"更好"**：P2 的超越根本不靠注视，且 P5 已显示换更准的注视目标
在知觉层无收益（mainline §5-1）。所以 P6 不赌"PSSP 让系统更好"，而问"**能動的注意的收益
在哪类场景才出现**"——P1-2 的单人为主商场里它不是瓶颈；多人竞争性场景才可能是。
即便 null，也是对边界的干净回答。

**★ P6 的立论基础：两个轴与两条机制**（被问"P5 都 null 了，P6 还做什么"时的答案）

先分清两个轴——之前叙事一直在它们之间滑动：

| 实验 | 什么在变 | 运动 | 场景 | 测什么 | 结果 |
|---|---|---|---|---|---|
| P4 | 目标＋运动（捆绑） | 随条件变 | 无后果游戏 | 知觉 GQS/GEQ | 有差（混淆） |
| P5 | 只有目标 | 共通（控制） | 无后果游戏 | 知觉 GQS/GEQ | 无差 |
| P6 | 只有目标 | 共通（控制） | **有后果** | **任务成果（配布率）** | ？ |

- **P4→P5** 只动了"运动 捆绑→控制"（场景/测量不变）→ 回答**运动 vs 目标**（在知觉通道）：
  运动主导知觉，目标不主导。
- **P5→P6** 只动了"场景 无后果→有后果、测量 知觉→任务成果"（运动仍控制、目标仍是被改的量）
  → 回答**有没有后果**。
- 所以 **P6 关注的是"后果"，运动是控制变量**，不是研究对象（运动问题 P4→P5 已答完）。

**为什么 P6 不是"换个因变量的 P5"（否则也会 null）——知觉与任务是两条不同机制：**

- 知觉通道（P5 测）：改目标 → 人**觉得**机器人更好吗？→ 不会（运动才影响观感）。
- 任务通道（P6 测）：改目标 → **结果**变吗（谁拿到くじ）？→ 机制不同：看向/招呼 A → A 更
  可能停下接くじ；看错 → 该接的人走了。这是**注意协调**效应，非观感效应。
- 故 P5 的"目标不影响知觉"**不蕴含**"目标不影响任务成果"。**P5 关掉知觉通道，P6 打开任务通道**
  ——这就是 P6 的合法性与不重复性。

**主张纪律**：P6 检验的是**目标选择的"协调型"任务价值**，与其在 P5 被否的知觉价值
**分属两条机制**；P5 与 P6 不矛盾，是同一变量在两条通道的两次测量（一负、一待测）。
（在 proposal2 的能動性框架下，这条"任务价值"进一步限定为**边界刻画**——见 §1、§5。）

## 2. 与 P1-2 的范式对应（沿用，proposal2 里意义更强）

范式逐项复刻 P1-2 消毒液实验（场所/自由来去/七动作库/自动成果事件/外生曝露量），
唯一新变量＝**注意目标的选择时机**（C1 反応的 vs C2 先読み）。详见对照表：
[../proposal1/p6-kujibiki-design.md](../proposal1/p6-kujibiki-design.md) §2。

在 proposal2 里这个复刻不只是"像 P1-2"，而是**字面回到 P1-2 的任务类型**去解它当年没解的注意问题——
所以七动作库里的 **TRACK**（P1-2 只会"看最近的人"）正是被 P6 升级的对象：
**C1/C2 用声音+（预测）判断该看谁，替换掉 P1-2 的"看最近脸"**。这是 P6 与 P1-2 最实的接点。

## 3. 核心实验设计（与 proposal1 同，摘要）

- **变量**：注意目标的选择时机。**C1 反応的（DoA，当前音图）** vs **C2 先読み（PSSP，预测音图）**；
  C3 Random 可选。下游运动控制器全条件共通（P5 shared framework 移植）。
- **行动层**：P1-2 七动作状态机，全条件钉死；状态机跟随"注意层当前指向对象"运行——
  **看谁→招呼谁→谁停下→谁抽签**，注意由此获得任务后果。
- **主指标**：**配布率 = 取签成功数 ÷ 通過者数**（外生分母）。成果事件＝出签口传感器触发
  （与 P1-2 消毒液喷头同型，仪器化无需新方法）。
- **成立性关门（最优先）**：安静单人路人对 PSSP/DoA 都无声信号 → 优势时刻假说限定在
  **多人/有声/结伴接近**场面（升为 primary planned contrast）；预备实验做 go/no-go：
  C2 现场是否真"先转"。
- **仲裁（engagement lock）**：进入 ENGAGED 后 TRACK 锁定当前对象至 SUCCESS/LEAVE，
  期间只允许受限"会釈瞥视"——防 C2 自我打断。
- **功效**：路人级二值 GLMM；预备实验必须估通過者/h 与基础配布率、算 MDE。

> 以上每项的详细论证（分母外生化、漏斗指标、计测系、限界）与 proposal1 设计稿一致，
> 不重复；见 [../proposal1/p6-kujibiki-design.md](../proposal1/p6-kujibiki-design.md) §3–§11。

## 4. proposal2 特有：连接"能動的注意 → 任务成果"的中间量

能動的注意要在任务层有意义，得先在**机制层**看到"注意得当"。P6 额外报告一个
P1-2 无法测、P6 才能测的中间量，作为边界刻画的证据：

- **误认代价（misattribution cost）**：机器人**先んじて**招呼了 A，但实际接近/欲互动的是 B
   的发生率，及其对配布率的影响。P1-2 的"看最近脸"应误认率最高；C2（声音+预测）若真选对，
   误认率应最低。
- 作用：把"能動的注意选对了人"这一机制层事实，接到任务成果上——**这正是 §5-1 说的
   "边界刻画"的可观测锚点**（误认率在多人场面才明显 → 收益也应集中在多人场面）。
- 需要 ground truth：谁是"真正的相手"在现场较难标注（不像 word wolf 有说话者标签）。
  → 预备实验需定义可操作代理（如：最终接受くじ者＝事后相手；或人工编码抽样片段）。
  **这是 proposal2 框架对 P6 的额外设计负担，也是它的额外回报。**

## 5. 对结果正负的两面准备（边界刻画，对正负都成立）

- **C2 > C1（配布率）**：能動的注意（声音+预测选对相手）在多人有后果场景**确有任务价值**
  → 边界＝"多人竞争性场景是能動的注意的收益区"；P1-2 的注视盲点可由 P3 路线在此类场景修复。
- **无差**：结合 P5 → "连这类多人现场，能動的注意也不是任务瓶颈" → 边界结论＝
  "能動的注意的收益区比设想更窄，别处（存在感/运动/流程）才是瓶颈"。同样是**清晰的边界回答**。
- **保底贡献**（不依赖方向）：把能動的行動＋能動的注意**统合进一台机器人、装回 P1-2 类现实任务
  并跑通**＋误认代价这一新指标＋行为监查现场移植。
- **注意（诚实）**：主张限定为"能動的注意の境界"，**不**声称"PSSP 让 P2 更好"（mainline §5-1）。
