# P6 くじ引きロボット実験 設計草案

> 博论 banner ④（統合＋帰結のある現場検証）的落地设计。
> **设计原则：不新造范式——直接继承 P1-2 酒精（消毒液）实验的已验证范式，
> 只插入一个新变量（注意通道）。**
> 主线文档：[mainline-notes.md](mainline-notes.md)。
> 最后更新：2026-07-15（教授视点评审 K1-K5 已反映：新增 §3 成立性关门、§5 engagement lock、
> §9 统计功效、§11 限界）

## 1. 一句话

> 公共空间通道旁的くじ引きロボット：路人自由来去，机器人主动招呼、在多人之间分配注意、
> 劝诱抽くじ；**任务成果＝くじ配布率（自动记录）**。
> 实验范式逐项复刻 P1-2 的消毒液推广实验，唯一新变量＝**注意目标的选择时机**
> （C1 反応的 vs C2 先読み/PSSP）。

---

**★ P6 的立论基础：两个轴与两条机制**（被问"P5 都 null 了，P6 还做什么"时的答案）

先分清两个轴——之前叙事一直在它们之间滑动：

| 实验 | 什么在变 | 运动 | 场景 | 测什么 | 结果 |
|---|---|---|---|---|---|
| P4 | 目标＋运动（捆绑） | 随条件变 | 无后果游戏 | 知觉 GQS/GEQ | 有差（混淆） |
| P5 | 只有目标 | 共通（控制） | 无后果游戏 | 知觉 GQS/GEQ | 无差 |
| P6 | 只有目标 | 共通（控制） | **有后果** | **任务成果（配布率）** | ？ |

- **P4→P5** 只动了"运动 捆绑→控制"（场景/测量不变）→ 回答**运动 vs 目标**（在知觉通道）：
  运动主导知觉，目标不主导。
- **P5→P6** 只动了"场景 无后果→有后果、测量 知觉→任务成果"（运动仍控制、目标仍是被改的量）
  → 回答**有没有后果**。
- 所以 **P6 关注的是"后果"，运动是控制变量**，不是研究对象（运动问题 P4→P5 已答完）。

**为什么 P6 不是"换个因变量的 P5"（否则也会 null）——知觉与任务是两条不同机制：**

- 知觉通道（P5 测）：改目标 → 人**觉得**机器人更好吗？→ 不会（运动才影响观感）。
- 任务通道（P6 测）：改目标 → **结果**变吗（谁拿到くじ）？→ 机制不同：看向/招呼 A → A 更
  可能停下接くじ；看错 → 该接的人走了。这是**注意协调**效应，非观感效应。
- 故 P5 的"目标不影响知觉"**不蕴含**"目标不影响任务成果"。**P5 关掉知觉通道，P6 打开任务通道**
  ——这就是 P6 的合法性与不重复性。

**主张纪律**：P6 检验的是**目标选择的"协调型"任务价值**，与其在 P5 被否的知觉价值
**分属两条机制**；P5 与 P6 不矛盾，是同一变量在两条通道的两次测量（一负、一待测）。

| 行动库 | 言语 HELLO/SANITIZER/THANKS/AWAY ＋ 非言语 TRACK/INITIALIZE/WAIT | 言语 HELLO/**KUJI**/THANKS/AWAY ＋ 非言语 TRACK/INITIALIZE/WAIT | 只把 SANITIZER 换成 KUJI（劝诱语），七动作结构原样 |

**博论逻辑（一句话）**：P1-2 在此范式下证明了"**行動の能動性**"产生任务成果；
P6 在**同一范式**上、把行动层钉死，检验"**注意の先読み性**"是否产生任务成果。
范式已被验证 ＋ 策略级单一新变量 → 归因干净，且统合（Field A 插进 Field B 的范式）是字面意义的。

## 3. 成立性の関門（最优先验证——K1）

**问题**：PSSP 预测的是**声源**。word wolf 里发话连续不断；商场走廊里大量路人**不出声**。
对无声的单独路人，PSSP 与 DoA 都无信号，C1 vs C2 在这些时段**无差可测**——
若无声单人场面占绝大多数，实验将在大部分时间空转。这是**成立性问题，不是适配问题**。

**对策（三件套）**：

1. **优势时刻假说的明确化**：C2 的期待差集中在
   (i) **边走边聊的群体接近**时（有声，可预测）、
   (ii) **多人应对中**（现对话产生声音；先"会釈"将到者 → 取りこぼし减少）。
   → 把"多人/有声场面"从事后分层**升级为 primary planned contrast**；
   "单人无声场面预期无差"本身写成明示预测——该层出 null 是**预测命中**，不是失败。
2. **声视融合的离线验证**：PSSP 本是 acoustic-visual 模型（P3），视觉通道原则上能从
   走近动作预测——但"模型是否实际利用视觉线索"**不可想当然**，用 Item 3 新数据
   离线检验"无声接近场景"的预测性能。
3. **预备实验的 go/no-go 关门**：现场 manipulation check——C2 是否真的比 C1 先转
   （応対の速さ的分布比较）。**先转不成立 → 不开主实验**，先回去解决模型/场景。

**场地选择连动**：群体（结伴聊天）比例高的场地对成立性有利——商业设施 > 安静走廊。

## 4. 三层控制架构（什么变、什么不变）

```
┌─ 注意层（WHO を見るか・いつ向くか）───────── ★実験変数★
│    C1: DoA（当前声源/接近者に反応）
│    C2: PSSP（予測された近未来の声源へ先回り）
│    → TRACK 动作的目标由此层供给；下游运动控制器全条件共通
│      （P5 の shared target-selection framework を現場に移植）
│
├─ 行动层（WHAT・WHEN）────────────────────── 全条件共通（钉死）
│    P1-2 七动作库的状态机（§5）；
│    状态机的输入 = 「注意层当前指向的对象」的状态
│    → 注意指对了人，HELLO/KUJI 才来得及 —— 帰結由此产生
│
└─ 对话层（想定外的应答）──────────────────── 全条件共通
     固定 teleop（本人）：只应答路人的自由提问；
     不控制头部、不触发动作（避免污染实验变量）
```

- HELLO/KUJI/THANKS/AWAY 用**预录/合成的固定语句**——P1-2 本来就是预定义言语动作，
  这里照搬；teleop 只兜底想定外对话（次数记录，作协变量）。
- **完全的"只变时机"分离不可能（K3，如实交代）**：切换频率等目标选择的派生量
  在条件间会残留（P5 实测：DoA P(switch)=0.081 vs PSSP 0.059）。运动控制器共通能钉住
  单次运动的质量，钉不住频率。→ 派生量用 P5 行为监查全程记录、作为协变量报告；
  主张限定为"**策略级比较**"（比较完整策略，而非纯时机分量）。
- 行动层实现二选一（§12 TODO）：**B 案（推荐）固定状态机**——规则透明、全条件严格一致；
  A 案（更像 P1）预备实验时由 teleop 示范、IRL 学轻量行动策略——学术上更美但工期风险大。
  **先按 B 案设计，A 案作为预备实验顺利时的升级项。**

## 5. 行动层状态机（全条件共通）

| 状态 | 迁移条件（自动检出） | 触发动作 |
|---|---|---|
| WAIT | 无人在半径 r 内 | 待机姿态；注意层自由扫描 |
| APPROACH | 注意对象进入 r 且朝向机器人 | **HELLO**（呼びかけ）＋ TRACK |
| ENGAGED | 对象停留 > t 秒 | **KUJI**（"くじ引きいかがですか"）＋指向出签口；**engagement lock 开始** |
| SUCCESS | 出签口传感器触发（签被取走） | **THANKS** → lock 解除 |
| LEAVE | 对象离开 r | **AWAY**（见送り）→ INITIALIZE → lock 解除 |

**Engagement lock（K2——注意通道与进行中交互的仲裁规则）**：

- 进入 ENGAGED 后，TRACK **锁定当前对象**直到 SUCCESS 或 LEAVE；
- lock 期间注意通道只允许**短暂会釈瞥视**（acknowledgment glance：<1s、频率上限、
  瞥后必回）指向新检出的对象——既保护进行中交互（**C2 不会自我打断**），
  又保留先読み的表达通道（先"点头示意"将到者＝多人场面取りこぼし减少的机制）；
- lock 规则**全条件共通**（含瞥视时长/频率上限）——条件间只差"瞥视指向谁、何时瞥"。
- 若无此规则，C2 可能中途弃掉眼前对象去追预测目标——实验测的就变成"打断策略"
  而非"先読み"，且 C2 会自伤。

多人场面的机制链：**看谁 → 招呼谁 → 谁停下 → 谁抽签**。C2 若能"先一步"指向将要
接近/将要说话的人，HELLO 就来得及；C1 只能事后追赶。
参数 r（接近半径）、t（停留阈值）由预备实验校准，全条件共用。

## 6. 条件与日程设计

| 条件 | 注意目标的决定 | 角色 |
|---|---|---|
| **C1 反応的（DoA）** | 当前声源/当前接近者 | 基线（事后追赶） |
| **C2 先読み（PSSP）** | 预测的近未来声源（+1s 等，用 Item 3 新模型） | 主条件（先于事件转向） |
| （C3 Random） | 与观测无关切换 | 可选锚点，按人流量与功效预算取舍（§9） |

- **时间块 between 设计**：每小时轮换条件、镜像 counterbalance
  （日1：C1→C2→C2→C1；日2：C2→C1→C1→C2），周末 2 日起步（天数由 §9 功效估算定）。
- **主计划对比（§3）**：多人/有声场面层的 C1 vs C2；单人无声层预期无差（明示预测）。

## 7. 平台与配布机构（Boxie 案，推荐）

- Boxie 无臂 → **手渡し不可**。解法：机器人旁置**出签口（くじ dispenser）**，
  机器人用头部朝向＋言语引导取签。
- **成功判定＝出签口传感器**（断光/微动开关）：签被抽走 → SUCCESS 自动记录。
  与 P1-2 的"消毒液喷头触发"完全同型——范式复刻里最有价值的一项：
  **成果事件的仪器化无需新方法**。
- child 型 humanoid（有手）为备选：手渡し更自然但需挂载确认＋工期风险；
  Boxie＋dispenser 为 9 月工期下的安全解。

## 8. 指标（全部自动记录；印象量表不作主证据——P5 教训）

**分母设计的关键原则（易踩坑）**：分母必须**外生**（条件无关）。
"进入半径 r 且朝向机器人"是**机器人行为的结果**——C2 若更早招呼会制造更多"接近者"，
分母被条件污染，配布率反而可能被拉低。→ **统计分母＝通過者数**（过线人数，由人流决定）；
"半径 r"只作状态机触发条件（行为层），与统计定义分家。

**漏斗结构**：

```
通過者（外生分母）
  └→ 立ち止まり     ← 誘引率：C2 优势假说作用点①（先转向→HELLO 及时）
       └→ ENGAGED（KUJI 劝诱到达）
            └→ SUCCESS（取签）＝主指标分子
```

| 层 | 指标 | 定义 |
|---|---|---|
| **主** | **配布率** | SUCCESS 数 ÷ **通過者数**（块内） |
| 副 | 誘引率 | 立ち止まり数 ÷ 通過者数 |
| 副 | 立ち止まり→取签转化 | SUCCESS ÷ 立ち止まり数 |
| 副 | 応対の速さ | 接近/发声 → TRACK 转向的时延（C2 应更早乃至为负） |
| 副 | 取りこぼし | 多人场面中未被 HELLO 而离开的人数（作用点②） |
| 检查 | 先読み性 manipulation check | tick 级注意日志 → P5 行为监查移植：确认 C2 确实"先转"（预备实验为 go/no-go，主实验续报） |
| 协变量 | teleop 介入次数、人流量、时间带、切换频率等派生量（§4） | 混合模型协变量 |

**归因链（每环有日志）**：先转向（応対の速さ）→ HELLO 及时（誘引率）→
KUJI 到达（ENGAGED）→ 取签（配布率）。有差时可定位在哪一环产生；无差时可报告链条哪环就断。

### 8.1 通過者数的计测系

1. **专用固定摄像头**（三脚架/壁挂广角俯拍通道）——**不可用机器人自身相机**：
   机器人相机随头转动，头的朝向正是实验变量，用它计数＝实验变量二次污染分母。
2. **检测＋跟踪＋虚拟过线计数**（YOLO＋ByteTrack 类；word-wolf-exp-eval utils 的
   人物检测代码可复用）。track ID 过线一次计一次；群体按个人计数；
   虚拟线全条件全时段固定同一条。
3. **人工校准闭环**（预备实验）：抽样时段人工点数 vs 自动计数，误检率目标 <5%；
   同时定规则：工作人员/回头客的排除标记、儿童的处理（兼顾伦理）。
4. **混乱度分层**用同一摄像头的同时 track 数判定（1 人 vs 多人），不新增设备。

## 9. 统计功效与分析（K4）

- **分析单位＝路人级二值结局**（取签 yes/no）的 **GLMM**：条件为固定效应，
  时间块・日・时间带为随机效应，人流量等为协变量——比块级比率检验功效高得多。
  漏斗各级（立ち止まり等）同样处理。
- **两个未知量决定一切**：通過者数/小时、基础配布率——**预备实验必须估计**，
  据此草算 MDE（最小可检出差）。
  例：基础配布率若只有 2-3%，两天人流可能远不够检出合理效应量——先算再开工。
- **功效不足时的选项**（按优先序）：选高人流场地 > 增加天数（2→4 日）>
  砍掉 C3 把时间让给 C1/C2。
- 多重比较：主对比只有一个（多人层 C1 vs C2），其余漏斗指标标注为 secondary/探索。

## 10. 对结果正负的两面准备（博论头部准则）

- **C2 > C1**：先読み注意在帰結成立时产生任务价值 → 矩阵右下格闭合为"有"。
- **无差**：结合 P5 → "此类轻交互中注意时机不是瓶颈" → 闭合为"无"，设计含意同样明确
  （资源应投向运动质量/存在感）。分层设计让 null 更可解释（§3：单人层 null＝预测命中）。
- **不依赖方向的保底贡献**：P1-2 范式上的统合系统现场运转实证＋行为监查的现场移植＋
  配布率指标的确立＋（若 A 案实现）IRL 行动策略在新任务的复用。

## 11. 事先交代的限界（K5——写进论文 limitation，别让委员先说）

1. **teleop（本人）对条件不盲**：知道当前模式，对话质量可能无意识偏移。
   对策：常见问题**标准话术表**、介入次数作协变量、对话内容事后编码检查条件间差；
   残余偏倚如实交代。
2. **现场声学环境**（P5 教训：空调占"他"标签大头）：HVAC/BGM 可能占据声学地图。
   → 预备实验含**场地声学勘查**（背景源的地图占用测定），必要时调整机器人位置/指向。
3. **くじ奖品的天花板/地板效应**：太诱人→人人都停（条件差被压缩）；太寒酸→无人理会。
   → 预实验调整奖品档次，目标把基础配布率调到中间带（利于检出差异）。
4. **策略级比较**（§4）：切换频率等派生量未独立操作——主张限定在策略整体，
   不声称纯"时机"分量的因果。
