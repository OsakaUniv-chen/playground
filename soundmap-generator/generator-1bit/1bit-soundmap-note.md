# 1bit音響カメラ（generator-1bit）実装メモ

## アルゴリズム（4手順）

1. **LUT作成**：16chマイクの配置と探索グリッド（FFT版と同一）から、マイクペアごとの音の到達時間差をサンプル数（整数）に丸めてテーブル化。
2. **BPF & 1bit化**：2000–8000Hzでゼロ位相バンドパスフィルタをかけ、符号だけを残して1bit信号に変換。
3. **ビットシフト＆XOR相関**：グリッド点ごとに、16マイクの全組み合わせ（C(16,2)=120ペア）についてLUTの遅延分だけ片方をビットシフトしてXOR、一致率を全ペアで平均してスコアとする（SRP-PHATに近い全ペア相関方式）。
4. **2Dマップ化**：スコアをFFT版と同じDelaunay補間で64×64のマップに変換。

## 実装：ビットパッキングによる高速化

64サンプルを1つのuint64語にパッキングし、XOR＋popcount（SWARビット数え上げ）を語単位（64サンプル/回）で処理することで、120ペア全相関でもCPU上で**~15ms/map**を実現している。CPU対CPUでpytorch版FFTビームフォーマー（~18ms/map）より高速。

## 精度の比較（実データ・G11_game4_DoA、189秒）

| | 一致率 |
|---|---|
| 全体 | 88.0% |
| 発話中（VAD有効） | 96.4% |

不一致の大半は「誰も話していない時の環境ノイズへの反応の違い」で、実運用上あまり意味を持たない。発話中の不一致は主にTeleoperator/Right境界付近（隣接領域）の空間分解能に起因する。

合成信号での検証では、単一音源の定位精度はSNR -20dB程度までFFT版とほぼ同等。二人が同時発話する場面では、1bit方式は弱い方の話者をFFT版より約6dB早く「かき消して」しまう傾向がある（信号の符号だけを見る非線形性による）。
