## 1. 背景：现在处于什么局面

### 1.1 Word Wolf（P5）的结论

- PSSP 的 +1 秒话者预测，即使在"已知左右二选一"下也只有 **54.8%**（随机基线 50%）；Tele（遥操作者）的动作更像 **DoA 追随（反应型）**，不是预测型。
- 系统有硬伤：Tele 音频是**单声道**（操作者无法靠听觉判方向）、座位/头向**系统性右偏**，破坏了条件间比较。
- **结论**：现行 Word Wolf **不再追加实验、也不投稿**。残余价值＝把问题分析透 + 数据转为训练数据 + 沉淀为下一实验的"bad know-how"。

### 1.2 导师在会上的关键判断（决定方向）

1. **否决"抽签（くじ引き）机器人"**：与前作 sanitizer 差分太小；用 PSSP/音频缺乏必然性（"为什么要预测声源？预测了又凭什么决定行为？"）；抽签没有真实社会需求。
2. **"没时间所以不用现代手法"不成立**：手法选择要从研究必要性论证，中村先生、石黒先生的审查视角都过不去。（我方的实际立场见下 §2.2：不是"没时间"，而是**数据/算力约束下走 finetuning 不现实**，因此用**预训练 VLM、不做 finetuning**。）
3. **焦点要从"PSSP 预测多准"转向"传感器输入 → 机器人行动决策"**，做一个"能让人相信可以社会实装"的系统；D 论前最好再投一篇系统实现论文。
4. 布置任务：**7/22 前发 3–4 个提案**，从两个角度各 2 案：
   - (a) 把 **acoustic map（音响地图/音图）**作为新特征量加入现代 **VLM** 架构，能带来什么具体"好处"；
   - (b) 存在哪些任务，"**预测下一个声源位置**"本身就直接产生价值。

### 1.3 导师给的系统像（叩き台）

- 输入：现场/操作者发话的**识别结果（language，非原始音频）** + 含 acoustic map 的**现场场景信息（vision）**。
- 输出：是否发话 / 是否做手势 / 注视对象等，用 **0/1 的 JSON** 表达行为决策。
- 贡献核心：把过去**由规则决定**的部分改成**学习式行为决策**，并在其中**证明音响信息的价值**。
- 注：这个"输入图像+语言、输出 0/1 JSON 行为决策"的系统像，正好可以用**预训练 VLM 直接 prompt** 实现，不必训练一个 action 头，与我方立场一致。

### 2.2 现方针：P6 沿"更好的能动行为决策系统"继续，用预训练 VLM

**关于手法：用 VLM，不做 VLA/finetuning**

- **VLA 不现实**：真正的 VLA（输出 action、需要 finetuning）需要大量数据和算力，对我剩余时间不现实。

- **P1-2 的弱点正好是 P6 的动机**：P1-2 的行为决策**只用视觉**（TRACK ＝只看"最近的脸"，不是说话者；**口罩会让视觉失效**）。多人场景里"该应对谁"需要**非视觉线索**。
- **P3 的音响地图作为模态补进来**（但不是"预测未来"的 PSSP，而是作为**当前场景表征**的一部分，和 V、L 并列）。
- **P6 ＝ 把 P1-2 的行为决策升级为 VLM 学习式**：输入＝语言（发话识别）+ 视觉 + 音响地图；输出＝发话/手势/注视/移动的行为决策；把原来规则决定的部分改成（预训练 VLM 的）学习式，并证明**音响地图能补上视觉+语言解决不了的对象选择**。

## 5. 提案候补（brainstorm）

### 5.0 核心难题：怎样的场景里"音图必然"？【所有候补的取舍闸门】

**问题**：sanitizer 型接客里，人**接近阶段**图像基本够用；音图只在"多人已围在机器人面前"时才有用——这让音图像"锦上添花"，贡献不足。要成立，任务必须让"音图必然"从**偶发情形变成任务核心**。

**三类"视觉根本做不到 / 做不好"的必然性来源**（任务设计要围绕其一）：

1. **视野外 / 360°**（视觉**物理上**做不到）：相机 FOV 是锥形，麦克风阵列全向。有人从背后/侧后呼叫、或视野外发生声学事件，视觉根本感知不到，只有音图能定位。**最强的"vision 不可能"论据。**
   - 与 P4/P5 划清：这不是"预测未来说话者"，而是"对**当前**视野外声源做**行为决策**"。价值须落在**决策**（要不要中断眼前交互去应对新声源）上，否则退化成单纯 DoA 转头就不新了。
2. **多人皆可能交互、无单一代表的热闹群体**（视觉难绑定）：公共空间里成群走过/围拢机器人的人（典型如一群学生），人人都可能搭话、无固定代表；抢话 + 互相遮挡 + 朝向不定，视觉难把"这句话"绑定到"哪个人"。音图给出当前活跃说话者方位来消歧。
   - **反例（已否决）：医院口罩受付**——虽全员戴口罩，但①院内放机器人许可难拿，②多为"一组一代表"，机器人只需应对代表，多人歧义并不成立。→ 口罩≠好场景，**关键在"人人皆可能交互"**。

**设计原则**：让"多人/视野外歧义"成为**任务常态**，且**判错有真实代价**——音图才从"锦上添花"变成"缺了就系统性出错"。

### 5.1 本线候补（角度 a，非移动，缺 §5.0 必然性者不留）

- **A-1 餐厅多人点单机器人**【企划见 §7.1】
  固定位（餐桌前或店门口排队时）。人人都要点单（天然"无单一代表、人人皆交互"），机器人须看向当前开口者、跟踪谁点了谁没点、决定下一个看谁。对话由远程操作者负责，机器人自主的是**注视/轮次决策**。"看 A 问 B"是直观失败模式；音图价值须靠 ablation 实测。
- **A-2 公共空间围拢案内机器人**【企划见 §7.2】
  固定位案内（车站/活动会场，不移动），被人群 360° 围住（含背后）。机器人须转向并回应当前开口者。音图定位超 FOV 的开口者——视觉物理上看不到。**诚实风险**：开放空间噪声多，难分"真求助"还是环境噪声，须设误触发判据。

## 7. 本线提案（角度 a）

### 7.0 企划书统一格式

每案按此六项写，直白、完整、无废话：

1. **名称 + 核心任务**：机器人名 + 一句话精准描述任务核心，附一句使用场景/社会需求。
2. **任务设计**：机器人要做的事，分点。
3. **手法**：动作与对话如何设计；分工（对话＝远程操作者 / 自主决策＝机器人）；VLM 的输入/输出如何管理（写清 prompt 喂什么、输出 JSON schema）；用到的其他模型写清；对照基线（rule-based）。
4. **轮次设计**：真实交互流程（一次交互从头到尾怎么走）。
5. **音图必然性**：为何此任务非音图不可（视觉/语言解不了的点）＋如何验证（ablation）。
6. **与 P1-2 的关联**：如何承接/补完 P1-2。

### 7.1 A-1 点单机器人

**1. 名称 + 核心任务**：**点单机器人**。餐桌前（或店门口排队）为多位客人逐个点单，核心＝**始终看向当前要点单的人**并推进点单。餐厅点单量大、可落地，多人"人人都要点"、无单一代表。

**2. 任务设计**
- 判断当前谁在开口点单，注视该人。
- 跟踪**各人点单是否完成**（含被他人代点，由对话内容判定，非"是否开过口"）。
- 无人主动时，选一名**未完成者**注视并催单。
- 全员点完则结束。

**3. 手法**
- 分工：**对话（说什么）＝远程操作者**；**注视目标 + 轮次动作＝机器人自主**。
- 感知：麦克风阵列→音图（各方向 DoA/强度）；相机→各座位客人位置与朝向。
- 决策＝**预训练 VLM（无 finetuning）**。输入＝`{各客人位置/朝向(vision), 当前音图活跃方位, 各人已点/未点状态, 最近发话识别文本(language)}`；输出 JSON＝`{look_at: 客人ID, prompt_order: bool, done: bool}`；prompt 描述上述规则。
- 对照基线：纯 rule-based（音图最强方向→look_at）。

**4. 轮次设计**
- 开场一句"准备好了就可以点单哦"，不指定谁先。
- 客人主动开口→音图定位→机器人看向他→操作者接单。
- 下一位主动→转向下一位。
- 一段时间无人主动→选一名未点者，注视＋"您这边看好了吗？"。
- 全员点完→结束语。

**5. 音图必然性**：多人同桌、朝向/遮挡不定，仅凭图像判"此刻谁在开口"不稳，易"看 A 问 B"；音图即时给活跃说话者方位，把这单话绑到人。**验证**：ablation（有/无音图）比注视对象正确率——餐桌前视觉或已够用，增益须实测。**风险（代点）**：家长代孩子点、一人代全桌点会退化为"单一代表"、音图价值下降；故目标场景锚在**多人各自点**（朋友/同事各点各的），代点须正确处理但非卖点。

**6. 与 P1-2 的关联**：P1-2 行为决策仅用视觉（TRACK＝最近的脸），多人点单下失效。本案把输入升级为 V+L+音图、决策交 VLM，是 P1-2 的直接补完。

### 7.2 A-2 围拢案内机器人

**1. 名称 + 核心任务**：**案内机器人**。车站/活动会场固定位，被人群 360° 围住时**转向并回应当前开口提问者**。公共空间常被成群、随意搭话的人包住，人人可能开口、无单一代表。

**2. 任务设计**
- 全向检测（含视野外）谁在向机器人开口。
- 判断是"真求助"还是环境噪声。
- 转向并回应当前开口者；多人先后开口依次处理。

**3. 手法**
- 分工：**对话＝远程操作者**；**转向目标 + 是否响应＝机器人自主**。
- 感知：麦克风阵列→全向音图（DoA）；相机→FOV 内人物。
- 决策＝**预训练 VLM（无 finetuning）**。输入＝`{FOV 内人物(vision), 全向音图活跃方位, 该方位发话识别文本(language)}`；输出 JSON＝`{turn_to: 方位角, respond: bool}`；"真求助"判据由 VLM 综合"语言内容像不像提问 + 音图 + 朝向"给出。
- 对照基线：纯 rule-based（音图最强方向→turn）。

**4. 轮次设计**
- 待机，全向监听。
- 某方位出现疑似向机器人的发话→VLM 判是否真求助→是则转向该方位（可能超原 FOV）→操作者应答。
- 应答中他处又有人开口→处理完当前再依次转向。

**5. 音图必然性**：开口者常在相机 FOV 之外（侧后），**视觉物理上看不到**，唯音图能定位。**诚实风险**：开放空间噪声多，难分真求助/噪声。**验证**：有/无音图 ablation，评价须报告漏应率与**误触发率**。

**6. 与 P1-2 的关联**：P1-2 只用视觉、锥形 FOV，围拢下失效。本案用全向音图补上 FOV 盲区、决策交 VLM，是 P1-2 的正当扩展。
