### 2.5 一致率（confusion matrix 的）解析
- **§3.4 のような集約分布どうしの比較ではなく、「同一観測」に基づく tick 単位の比較**にすべき。
  - 収集済み rosbag から DoA / PSSP / Random を**再実行**でき、各 rosbag に**遠隔操作者（Tele）の頭向き映像**が残っているため、**同じ観測に対する各方策の出力**を突き合わせられる。
- 出すのは **confusion matrix 的なもの（正答率ではなく「一致率」）**。

**信号の定義とラベル数**
- `DOA(t)`：t 秒後の DoA 実測値（＝ t 秒後の実発話者）。`DOA(0)` = 現在の発話者。**4 ラベル**（L/R/Tele/Others）。
- `PSSP`：PSSP の注視ラベル。**4 ラベル**（L/R/Tele/Others）。
- `TEL`（遠隔操作者）／`RAND`：**2 ラベルのみ（L / R）**。

**各ペアの作り方**
どのペア「A vs B」も手順は同じ：
1. 各 tick（0.25 秒）で、A のラベルと B のラベルを 1 つずつ取る。
2. **行＝A、列＝B の表**に数え入れ、**一致 tick＝行と列が同じラベルのセル**（L–L, R–R, …）、**一致率＝一致 tick / 全 tick**。これが「confusion matrix 的な表」。
3. **表のサイズは 2 種類だけ**：
   - **両方 4 ラベル**（PSSP・DoA どうし＝ PSSP vs DOA1／DOA0 vs DOA1）→ **4×4**（Tele/Others も含めて集計）。
   - **TEL か RAND を含む**（一方が L/R のみ）→ **L/R の tick に絞って 2×2**。相手（PSSP/DoA）が Tele/Others を出した tick は除外する。
- **TEL vs DOA(lag)** だけ形が特殊：DOA を lag ずらして 2×2 を作り、**lag ごとに一致率を 1 個** → 「lag–一致率」曲線。

**比較ペアと目的**
| 比較 | 表の形 | 目的 |
|---|---|---|
| PSSP vs DOA1 | 4×4 | 予測がうまくいっているか（PSSP は +1s 予測） |
| PSSP vs RAND | 2×2 | 学習の効果はあったのか |
| ↳（PSSP vs RAND 補足） | 2×2 | Random セッションの、かつ **PSSP 出力が L or R だった時刻のみ**抽出し、実際に出力した L/R 指令値の **2×2 マトリクス**を作成 |
| TEL vs PSSP | 2×2 | 類似度 |
| TEL vs DOA(0) | 2×2 | ベースライン |
| TEL vs RAND | 2×2 | マニピュレーションチェック |
| TEL vs DOA(lag) ラグ掃引 | 2×2（×lag） | 人間の動作の違い／追従ラグの特定：lag = **-1.0, -0.5, 0.0, 0.5, 1.0, 1.5, 2.0, 2.5, 3.0, 5.0, 10.0** 秒（0.0 = TEL vs DOA(0)） |
| DOA(0) vs DOA1 | 4×4 | 1 秒空くと平均的にどれだけ発話者が変わるか（ベースレート） |
| RAND vs DOA1 | 2×2 | ベースライン |
| RAND vs DOA0 | 2×2 | ベースライン |

**Limitation（議論に追加）**
- **ロボットの行動が人間の行動（発話を開始するか等）に与える影響を無視している**点を、議論に加える。

## 3. 対応方針（プラン）

### 3.1 → 行動解析の指標（メモ 1.1・TODO 2.5）
- **メモ 1.1（cosine 類似度）と TODO 2.5（一致率／confusion matrix）は同じ狙い**と理解：
  §3.4 の「集約分布どうしの比較」を、**同一観測に対する tick 単位の突き合わせ**に置き換える。
- **指標は離散ラベル（Left/Right/Tele/Others、主に L/R）の一致率で計算する。ヨー角（連続値）や cosine は用いない**（合意済み）。
- 実装方針：
  1. **各 rosbag の観測から 4 方策すべて（PSSP / DoA / Random / Tele）を再導出**し、tick ごとの注視ラベルを生成。
     各 rosbag に**遠隔操作者の映像が残っている**ため、どのセッションでも `TEL` を同一観測上で再現でき、全方策を同じ観測で突き合わせられる（合意済み）。
  2. 2.5 の各ペアについて **一致（confusion）行列**（4×4、または L/R に絞った 2×2）と、**一致率（一致 tick／全体）**を算出。
  3. あわせて **4 ラベルの precision/recall**（メモ 1.1 後半の指摘）も付表として出す。
- **【要相談 D：PSSP vs RAND 2×2 の解釈】** 作り方は確定：**PSSP 出力が L/R の tick に絞り、行＝PSSP の L/R × 列＝RAND の L/R** の 2×2。RAND は観測に依らない乱数なので一致率は偶然（≈50%）になるはず。これを「PSSP は乱数を再現しておらず学習が効いている」とどう論じるか、意図を次回確認する。
- 実装メモ：着手前に、PSSP/DoA が rosbag から再現できることを小規模に sanity check する。
