# 1) torch (cu124 专用 index; 驱动 570/CUDA12.8 兼容):
#    pip install torch==2.6.0 torchvision==0.21.0 --index-url https://download.pytorch.org/whl/cu124

# 2) autoawq 与配套内核 (必须 --no-deps, 否则会把 torch 拉成不兼容的 cu13 版):
#    pip install --no-deps autoawq==0.2.9
#    pip install --no-deps autoawq-kernels==0.0.9

# 3) 其余 (普通 index):
Pillow
accelerate
qwen-vl-utils
hf_transfer                # HF 下载加速(设 HF_HUB_ENABLE_HF_TRANSFER=1)
transformers==4.51.3       # 必须钉住: <5(需更旧 torch 的 DTensor) 且 >=4.49(Qwen2.5-VL)

# echo 后端只需 Pillow, 上面 AWQ 相关可跳过。

# ---- 已验证 (2026-07) --------------------------------------------------------
# torch 2.6.0+cu124 (cuda available=True, RTX 3090), transformers 4.51.3,
# autoawq 0.2.9 + autoawq-kernels 0.0.9。32B-AWQ 加载 ~19s, 推理 ~5.5s/张。
#
# ---- 关键坑 (都已在 vlm_server.py 里处理) -----------------------------------
# 1) 该 checkpoint 的 lm_head 其实未量化(存 fp16 lm_head.weight), 但其 config 的
#    modules_to_not_convert 只有 ['visual'] 漏了 lm_head -> transformers 误量化、
#    丢弃真权重 -> 推理报 "expected Int but found Half"。
#    修复: 加载时把 'lm_head' 补进 modules_to_not_convert (server 里已做)。
# 2) AWQ CUDA 内核只吃 float16 -> 加载用 torch_dtype=torch.float16 (server 里已做)。
#
# ---- 旧驱动 (cu118) 为何不行 -------------------------------------------------
# 驱动<=525 只能 cu118 -> autoawq-kernels 最高 0.0.7, 其 gemm ABI 与 Qwen2.5-VL
# 所需的 autoawq>=0.2.7 不兼容; Triton 回退内核也无法在现代 triton 上编译。
# 结论: 跑 32B-AWQ 需要驱动>=570 + 上面的 cu124 组合。
